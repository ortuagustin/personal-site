Vue: Componentes privados con JSX

En general, creo que la mejor manera de escribir aplicaciones Vue es usando Single File Components (SFC). Escalan bien en proyectos pequeos, medianos y grandes, podemos usar Scoped CSS (bsicamente, CSS con alcance lmitado a dicho componente), son muy intuitivos y fciles de usar, entre otras.

Algo que me gustara que tengan es la posibilidad de que, del mismo modo que existe el Scoped CSS, poder tener subcomponentes, que sean privados a dicho SFC en cuestin. Por el momento, esto no es posible en un SFC, pero se pueden usar render functions.

En realidad, un SFC simplemente es una forma ms amigable de escribir estas funciones de renderizacin, ya que estos son luego traducidas a render functions cuando se construye la aplicacin.

Se pueden "mezclar" los diferentes approachs sin ningn problema, es decir, podemos tener SFC y render functions coexistiendo sin ningn problema

Si tomamos el ejemplo de la documentacin, vemos que se define, usando solo JS, un componente con un mtodo render, que es el que, valga la redundancia, define como se renderizar el mismo:

Vue.component('anchored-heading', {
  render: function (createElement) {
    return createElement(
      'h' + this.level,   // tag name
      this.$slots.default // array of children
    )
  },
  props: {
    level: {
      type: Number,
      required: true
    }
  }
})

El problema es que, obviamente, la forma de escribirlos es de muy bajo nivel, y adems de engorrosa, podra ser hasta dificil para componentes no triviales como el ejemplo de arriba.

Existe una alternativa, JSX que nos brinda una API de mayor nivel para escribir estas funciones.

JSX

No vale la pena entrar en detalle de qu es JSX, ya que la red est plagada de informacin al respecto. Con saber que simplemente es una manera de escribir render functions usando una API de mayor nivel de abstraccin, es suficiente. Yo no la considero tan amigable como la de los SFC, pero en esas situaciones en las que se desea extraer un pequeo subcomponente, me resulta muy til.

Entonces, la idea es poder hacer algo cmo esto:

<template>
  <div>
    ...
    <Subcomponent text="foo" @clicked="bar" />
  </div>
</template>

<script>
const Subcomponent = {
  props: {
    text: { type: String, required: true },
  },

  render() {
    return (
      <button type="button" onClick={() => this.$emit('clicked')}>
        {this.text}
      </button>
    )
  }
}
</script>

Es interesante destacar que JSX es muy dinmico, ya que bsicamente estamos escribiendo JS, y podemos usar cualquier cosa que nos provee el lenguaje. Por ejemplo, el spread operator, hace que pasar un objeto como propiedad sea mucho ms sencillo.

En proyectos creados con Vue CLI, ya tenemos todo lo necesario para poder empezar a usar JSX. Si no es el caso, en ste repositorio se explica qu es lo que se debe instalar.

Cheatsheet Vue <-> JSX

Si bien son similares, hay algunas cuestiones que en las que el paso de un SFC a JSX no es tan directo. Por eso decid armarme un cheatsheet para tener de referencia para cuando no recuerdo cmo escribir algo en JSX.

Nota: en el repositorio citado anteriormente hay ms ejemplos.

Contenido

Lo ms bsico y elemental podra ser renderizar algn contenido que est en variables del modelo/propiedades:

SFC:

<template>
  <div>
    <p>{{ foo }}</p>
    <p>{{ bar }}</p>
  </div>
</template>

<script>
export default {
  props: {
    foo: { type: String, required: true },
  },

  data() {
    return {
      bar: 'bar'
    }
  },
}
</script>

JSX:

const Example = {
  props: {
    foo: { type: String, required: true },
  },

  data() {
    return {
      bar: 'bar'
    }
  },

  render() {
    return (
      <div>
        <p>{this.foo}</p>
        <p>{this.bar}</p>
      </div>
    )
  }
}

Es similar a un SFC, solo que en lugar de {{ var|prop }}, es {this.var|prop}

Binding

En un SFC usamos v-bind: o la forma abreviada : para bindear el valor de una variable o propiedad computada:

<template>
  <div>
    <UserProfile :user="user" />
  </div>
</template>

<script>
export default {
  data() {
    return {
      user: {
        name: 'John Doe',
      },
    }
  }
}
</script>

Y el componente UserProfile:

<template>
  <div>{{ user.name }}</div>
</template>

<script>
export default {
  props: {
    user: { type: Object, required: true },
  },
}
</script>

En JSX, el componente UserProfile sera as:

const UserProfile = {
  props: {
    user: { type: Object, required: true },
  },

  render() {
    return (
      <div>{this.user.name}</div>
    )
  }
}

Eventos

En un SFC usamos v-on:, o la forma abreviada @ para asignar un manejador a un evento. Por ejemplo:

<template>
  <button type"button" @click="search">Search</button>
</template>

<script>
export default {
  methods: {
    search() {
      ...
    }
  }
}
</script>

En JSX:

const Example = {
  render() {
    return (
      <button
        type="button"
        onClick={() => this.search}
      >
        Search
      </button>
    )
  },

  methods: {
    search() {
      ...
    }
  }
}

En este caso utilic un callback, aunque sto no siempre es necesario. En particular, ste ejemplo tambin es vlido de la siguiente forma:

const Example = {
  render() {
    return (
      <button type="button" onClick={this.search}>Search</button>
    )
  },

  methods: {
    search() {
      ...
    }
  }
}

Es simplemente una convencin, dado que en general, cuando manejamos eventos, es necesario acceder a algn parmetro que el evento enva, entonces:

const Example = {
  render() {
    return (
      <input type="text" onChange={event => this.handler} />
    )
  },

  methods: {
    handler(event) {
      console.log(event.target.value)
    }
  }
}

Binding de class y style

Tal y como se menciona en la documentacin, algo muy comn es tener que modificar los atributos class y style de un elemento HTML; es por eso que Vue provee algunos mecanismos especiales para v-bind cuando se trata de estos atributos.

  • Object Syntax

sta sintaxis nos permite utilizar un objeto para especificar que clase queremos que se active/desactive (toggle, en la jerga) cuando una determinada propiedad evalua a true. El ejemplo de la documentacin:

<template>
  <div v-bind:class="{ active: isActive }">...</div>
</template>

A m en particular me gusta utilizar la forma abreviada de v-bind. Tambin prefiero poner la clase, en este caso, active, como string, debido a que de esa manera es ms claro:

<template>
  <div :class="{ 'active': isActive }">...</div>
</template>

Basicamente, si la propeidad isActive evalua a true, Vue agregar la clase "active" al div.

Por supuesto que podemos usar un objeto con mltiples propiedades:

<template>
  <div
    :class="{ 'active' : isActive, 'text-red-500' : hasError }"
  >
    ...
  </div>
</template>
  • Array Syntax

sta forma nos permite emplear un arreglo para aplicar una lista de clases:

<template>
  <div :class="[activeClass, errorClass]">...</div>
</template>

<script>
export default {
  data() {
    return {
      activeClass: 'active',
      errorClass: 'text-red-500'
    }
  }
}
</script>

Lo cual genera el siguiente HTML:

<div class="text-red-500 active">...</div>

Tambin es posible utilizar el operador ternario para activar alguna clase de forma condicional:

<template>
  <div :class="[isActive ? 'active' : '', errorClass]">...</div>
</template>

De sta manera, si isActive evalua a true, entonces aplica la clase "active", caso contrario, como la expresin devuelve un string vacio, no aplica nada; y luego la clase evalua la expresin errorClass la cual devolver alguna clase derivada de una propiedad del data o computed

Notar que si tenemos que manejar una sola clase, es posible utilizar el operador ternario sin necesidad de tener el array:

<template>
  <div :class="isActive ? 'active' : ''">...</div>
</template>

Por ltimo, tambin destacar que cuando bindeamos ":class", Vue no "pisa" el atributo "class" del elemento HTML, sino que los concatena. Es decir, modificando un poco el ejemplo anterior:

<template>
  <div class="card" :class="isActive ? 'active' : ''">...</div>
</template>

Genera el siguiente HTML, si isActive evalua a true:

<div class="card active">...</div>

Y, si isActive evalua a false:

<div class="card">...</div>

Es decir, la clase "card" siempre est

Binding de class y style en JSX

Ahora bien, algn mecnismo similar, "especial", para llevar stos ejemplos a JSX?. Bueno, la realidad es que no tenemos nada "especial". La nica manera es usando el binding comn y corriente de JSX, interpolando y usando template strings:

<div class={`${isActive ? 'active' : ''}`}>...</div>
<div class={`active ${hasError ? 'text-red-500' : ''}`}>...</div>

Componentes

Por supuesto que un componente escrito con JSX puede contener a otros componentes.. independientemente de si son JSX o SFC. Hay que recordar que simplemente son diferentes APIs, pero que en el fondo, todo se reduce a render functions.

En SFC:

  • Parent:
<template>
  <div>
    <Child />
    <Child />
  </div>
</template>

<script>
import Child from './Child'

export default {
  components: { Child },
}
</script>
  • Child:
<template>
  <div>...</div>
</template>

Traducido a JSX:

const Child = {
  render() {
    return (
      <div>...</div>
    )
  }
}

const Parent = {
  render() {
    return (
      <div>
        <Child />
        <Child />
      </div>
    )
  }
}

Notar que no es necesario especificar en Parent components, que componentes hijos va a tener el componente padre, como s es el caso en SFC

Componente dinmico

En Vue, tenemos un componente especial, el componente dinmico.

En SFC, sera algo as:

<template>
  <div>
    <Component :is="icon" class="...">
  </div>
</template>

<script>
const icons = {
  foo: 'FooIcon',
  bar: 'BarIcon',
}

import FooIcon from '@/assets/icons/foo.svg'
import BarIcon from '@/assets/icons/bar.svg'

export default {
  components: { FooIcon, BarIcon },

  props: {
    type: { type: String, required: true },
  },

  computed: {
    icon() {
      return icons[this.type]
    }
  }
}
</script>

En JSX, es completamente diferente, ya que directamente no tenemos este component especial. Por lo tanto, un equivalente, sigue el siguiente patrn:

import FooIcon from '@/assets/icons/foo.svg'
import BarIcon from '@/assets/icons/bar.svg'

const Example = {
  props: {
    type: { type: String, required: true },
  },

  render() {
    const icons = {
      foo: 'FooIcon',
      bar: 'BarIcon',
    }

    // equivalente a <Component :is="icons[tipo]" />
    const Component = icons[this.type]

    return (
      <div>
        <Component class="...">
      </div>
    )
  }
}

El nombre de la constante Component es arbitrario, podra ser cualquier cosa

Loops

En SFC utilizamos la directiva v-for para recorrer una coleccin y renderizar una lista de elementos:

<template>
  <div>
    <ul>
      <li v-for="item in items" :key="item">
        {{ item }}
      </li>
    </ul>
  </div>
</template>

<script>
export default {
  data() {
    return {
      items: [
        'foo',
        'bar',
      ]
    }
  }
}
</script>

En JSX, debemos usar JavaScript para loopear. Una forma es el mtodo map de Array

const Example = {
  data() {
    return {
      items: [
        'foo',
        'bar',
      ]
    }
  },

  render() {
    return (
      <div>
        <ul>
          {this.items.map(item => (
            <li>{item}</li>
          ))}
        </ul>
      </div>
    )
  }
}

Notar que no es necesario indicar el atributo key

Condicionales

En un SFC se utiliza la directiva v-if:

<template>
  <div>
    <button type="button" @click="deleteItem" v-if="hasAccess">Delete Item</button>
  </div>
</template>

<script>
export default {
  props: {
    user: { type: Object, required: true },
  },

  methods: {
    deleteItem() {
      ...
    }
  },

  computed: {
    hasAccess() {
      this.user.isAdmin
    },
  },
}
</script>

En JSX, al igual que el caso de los loops, usamos JavaScript:

const Example = {
  props: {
    user: { type: Object, required: true },
  },

  methods: {
    deleteItem() {
      ...
    }
  },

  computed: {
    hasAccess() {
      this.user.isAdmin
    },
  },

  render() {
    return (
      <div>
        {this.hasAccess && (
          <button type="button" onClick={() => this.deleteItem}>
            Delete Item
          </button>
        )}
      </div>
    )
  },
}

Es posible escribir algn condicional ms complejo:

const Example = {
  render() {
    return (
      <div>
        {(this.hasAccess || this.isDevelopment) && (
          <button type="button" onClick={() => this.deleteItem}>
            Delete Item
          </button>
        )}
      </div>
    )
  },
}

const Child = {
